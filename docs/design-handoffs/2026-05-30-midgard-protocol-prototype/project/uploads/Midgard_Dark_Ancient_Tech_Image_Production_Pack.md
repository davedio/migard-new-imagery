## Non-Negotiable Direction

The active visual model is Mythic Tree World with Dark Ancient Tech styling. The failure mode is a crude stick-frame tree, literal forest scene, or old colorful Tree World reuse.

Tree-world DNA should become ancient technical architecture:

- An axis-mundi-like trust engine.
- A canopy expressed as a stone/gold vault, topology, or proof aperture.
- Branching paths expressed as gold circuitry and verification rails.
- Root-like structure expressed as settlement bedrock, foundation channels, or L1 rails.

Use:

- Stone-and-gold proof gates.
- Ancient trust engines.
- Architectural world-tree topology.
- Organic Rootwork as technical topology, not literal nature.
- Settlement rails.
- Monolithic technical infrastructure.
- Gold light moving through carved channels.
- Midgard green as a controlled trust signal.
- Deep ink, basalt, obsidian, charcoal, muted gold, and small green activation points.

Avoid:

- Stick-frame tree diagrams, literal forests, wooden trunks, leaves, vines, natural bark, fantasy nature.
- Generic crypto: coins, chains, glowing cubes, token icons, blue-purple gradients.
- Fantasy-game staging: castles, characters, weapons, quests, monsters, temples full of fake lore.
- Obvious AI artifacts: fake text, pseudo-runes, impossible UI, melted geometry, inconsistent symbols.
- Any final text, logos, labels, or UI copy generated inside the image model. Add those later in Figma, PPTX, or code.

## Recommended Workflow

1. Generate still concept frames first.
2. Select 3 to 5 frames with the right architecture, lighting, and composition.
3. Upscale or re-render the best frame.
4. Clean artifacts manually in Figma, Photoshop, or equivalent.
5. Add logo, copy, labels, proof chips, and interface text manually.
6. Use the cleaned still as the source for motion tests.
7. Translate final geometry into code, Spline, Three.js, Rive, or PPTX shapes where the asset needs to be editable.

## Core Prompt Formula

Use this structure:

```text
Midgard dark ancient technology mythic tree world, [specific object], trust architecture expressed as basalt and obsidian infrastructure, architectural world-tree topology, muted gold proof circuitry, controlled emerald green trust signal, deep ink atmosphere, premium technical schematic detail, cinematic but restrained, architecture-first, no people, no text, no logo, no fantasy forest, no generic crypto, high contrast, precise geometry, original design
```

Add aspect ratio by output:

- Deck hero: `16:9`
- Website hero: `21:9` or `16:9`
- Mobile hero: `9:16`
- Social proof card: `4:5` or `1:1`
- Background texture: `3:2` or `16:9`

## Production Notes By Tool

Midjourney or equivalent:

- Use for still-image exploration only.
- Generate without text or logo.
- Iterate around composition, material, light, and visual metaphor.
- Prefer lower chaos for deck/product surfaces; use higher stylization only for early exploration.

Runway, Luma, or equivalent:

- Use only after selecting and cleaning a still frame.
- Keep motion subtle: light travel, slow push, slight parallax, signal activation.
- Reject outputs where symbols morph, fake text appears, or geometry melts.

Figma:

- Add Midgard logo, copy, UI labels, proof chips, safe zones, and responsive crops manually.
- Build reusable components for proof cards, status labels, readiness rows, and trust-layer panels.

Spline or Three.js:

- Use for interactive proof gate, settlement rail, or trust-layer topology.
- Keep geometry simple enough to render fast.
- Include static and reduced-motion states.

Rive:

- Use for lightweight stateful diagrams.
- Best for trust-layer selector, proof-card states, and readiness-track indicators.
