# Midgard Claude Design Instructions: Dark Trust Architecture

Prepared: 2026-05-27

Status: Paste-ready creative instructions for Claude Design or a design model.

## Purpose

Use this file to brief Claude Design, Figma Make, Claude Artifacts, or a human designer on the darker Midgard trust-architecture direction.

The goal is to produce high-quality visual concepts, layout systems, style frames, and interactive explainers.

## How To Use

1. Paste the Master Instruction Block into Claude Design as the project instruction.
2. Pick one Task Prompt and paste it as the current design assignment.
3. Ask for 2 to 3 directions first, not one final design.
4. Review against the QA checklist at the bottom.
5. Move the winning direction into Figma, PPTX, Rive, Spline, Three.js, or static asset production.

## Master Instruction Block

```text
You are designing for Midgard, a Cardano-native Layer 2 brand. Your job is to explore a darker, premium Mythic Tree World visual system with trust architecture as the mechanism layer.

Brand direction:
- Voice: Category Rebel / Confident Challenger.
- Tone: 70 direct / 30 defiant.
- Visual world: Dark Ancient Tech / Mythic Tree World.
- Mechanism layer: Trust Architecture.
- Core feeling: inevitable because the architecture earns trust.
- Tagline territory: Trust The Path.
- CTA territory: Verify Then Build.
- Public campaign territory: Secure Migration.
- Primary audience: builders and partners.
- Audience paths: Users, Builders, Partners.

Design posture:
- Make the system feel ancient, technical, premium, alive, and credible.
- Use mostly dark backgrounds with Midgard green, muted gold, stone, and deep ink.
- Use beautiful schematics, proof labels, paths, layers, and source/status chips.
- Use mythic visuals implicitly. Do not write lore. Do not rename navigation.
- Show trust architecture: activity, batching, proof, challenge, settlement, and Cardano L1 anchoring as visual layers.
- Simulated flows are acceptable if clearly treated as demonstration data.

Copy posture:
- Use short working labels only.
- Keep language direct, sparse, and useful.
- Let the visual system do most of the atmospheric work.

Visual rules:
- No generic crypto art.
- No coin stacks, glowing cubes, random chains, purple-blue crypto gradients, bokeh, or decorative orbs.
- No fantasy-game forests, castles, characters, weapons, quests, monsters, or obvious rune gimmicks.
- No obvious AI artifacts: fake text, melted geometry, inconsistent symbols, pseudo-interfaces, random pseudo-runes.
- Do not copy Roku City, named artists, named studios, or protected styles.
- Use Cardano blue only as a restrained ecosystem cue, not as a dominant palette.
- Avoid cyan as a default accent.

Design requirements:
- Create a premium, useful system that could support a launch deck, website gateway, homepage explainer, docs visuals, and partner materials.
- Make every key screen work as a static image first.
- Include reduced-motion and static fallback notes for any motion concept.
- Make mobile feel intentionally designed, not cropped from desktop.
- Use accessible contrast and legible type.
- Keep final design editable in Figma/PPTX where practical.
```

## Task Prompt 1: Cinematic Gateway

```text
Create 3 visual directions for a Midgard cinematic gateway screen.

The gateway should feel like a dark ancient technology threshold: a mythic world-tree trust path or proof gate emerging from deep ink, controlled Midgard green light, and muted gold proof signals.

Requirements:
- Minimal copy only: logo, one short headline placeholder, one CTA placeholder, and 2 to 3 status/source chips.
- The main image should imply a trust path, not explain it with paragraphs.
- The main object should be tree-themed, but it must feel like ancient technical infrastructure: stone axis, proof aperture, gold circuitry, branching verification rails, canopy-like topology, and settlement bedrock. Do not make a stick-frame tree, wooden tree, leafy canopy, forest, or fantasy-game set piece.
- Show where desktop and mobile safe zones would be.
- Include a static fallback frame and a reduced-motion note.

Deliver:
- 3 directions with names.
- 1 desktop frame per direction.
- 1 mobile frame per direction.
- A short rationale for each direction.
- A do-not-use list for anything that would make the direction feel generic or off-brand.
```

## Task Prompt 2: Homepage Living Trust Architecture

```text
Design a homepage / second-page interactive explainer for Midgard called Living Trust Architecture.

The experience should show a simulated action moving through trust layers:
1. User or app activity.
2. Batch.
3. Proof.
4. Challenge path.
5. Settlement.
6. Cardano L1 anchor.

Requirements:
- Use a mostly dark, premium technical interface.
- Use tree/topology, route, rail, layer, aperture, and path metaphors visually, but keep labels plain.
- Include audience entry points for Users, Builders, and Partners.
- Add an optional Explore mode where users can inspect key layers.
- Use Natural Reveal rather than forced guided scroll.
- Clearly label simulated data as demonstration data.
- Include reduced-motion behavior and static fallback.

Deliver:
- Desktop layout.
- Mobile layout.
- Layer inspection state.
- Motion notes.
- Component list.
- Accessibility and performance notes.
```

## Task Prompt 4: Audience Path System

```text
Design a three-path audience system for Midgard: Users, Builders, Partners.

The paths should feel like one coherent brand, not three separate microsites.

Requirements:
- Use plain labels: Users, Builders, Partners.
- Users path answers: Why trust it?
- Builders path centers: Verify Then Build and Midgard Builder Readiness.
- Partners path centers: readiness tracks first, co-marketing after approval.
- Each path needs progressive layers: simple explanation, proof object, deeper detail, next action.
- Keep copy as short working labels.
- Do not use lore names for audience paths.

Deliver:
- Navigation pattern.
- Audience path landing block.
- Progressive disclosure component.
- CTA cluster.
- Mobile treatment.
- Empty/loading/error states if the design implies dynamic content.
```

## Task Prompt 7: Design QA Review

```text
Review this Midgard design direction against the brand rules.

Check for:
- Does it feel like Dark Ancient Tech / Mythic Tree World without becoming fantasy-game?
- Does it look premium, technical, and alive?
- Is the copy plain, minimal, and useful?
- Are Users, Builders, and Partners represented plainly?
- Are proof labels, source labels, confidence labels, and demonstration-data labels clear?
- Are motion concepts optional, performant, and reduced-motion safe?
- Are there obvious AI artifacts?
- Does it avoid generic crypto visuals?
- Does mobile feel intentionally designed?

Return:
- Top 5 strengths.
- Top 5 risks.
- Exact edits to make next.
- A pass/fail score for brand fit, visual originality, accessibility, and production readiness.
```

## Design Tokens For Exploration

| Token | Value | Use |
|---|---|---|
| Deep ink | `#07120B` | Main dark canvas |
| Charcoal | `#101820` | Secondary dark panels |
| Midgard green | `#20BE43` | Living trust signal, active path, key action |
| Muted gold | `#B7791F` | Proof, caution, warmth, settlement cue |
| Mist line | `#D7E2D8` | Dividers, diagram lines, low-emphasis UI |
| Paper | `#F8FAF5` | Utility surfaces and source-heavy pages |
| Slate | `#475569` | Metadata, subdued text |
| Cardano blue | `#0033AD` | Restrained Cardano cue only |

Typography starting point:

- Display: Poppins or Unbounded, depending on final deck direction.
- Body: Inter.
- Technical labels: JetBrains Mono.

Spacing and shape:

- Keep cards and panels at 8 px radius or less unless the existing design system says otherwise.
- Prefer precise layers, rails, proof paths, and status chips over decorative cards.
- Do not put cards inside cards.
- Use dark space deliberately. Do not fill every surface.

## Production Notes

Recommended workflow order:

1. Master visual direction first.
2. Static frames second.
3. Design system components third.
4. Motion prototype fourth.
5. PPTX/Figma production fifth.
6. Final copy pass last.

Do not ask a design model to invent the final page and final hero art at the same time. Separate concept art, layout, component system, and copy refinement.

## QA Checklist

- The design is dark, premium, technical, and alive.
- It does not feel like generic crypto.
- It does not feel like a fantasy game.
- It does not copy a named style, artist, studio, or protected work.
- It keeps the older colorful Tree World direction separate.
- It uses Users, Builders, and Partners as plain audience paths.
- It includes Verify Then Build and Trust The Path only as brand territories, not as overused slogans.
- It labels simulated activity as demonstration data.
- It includes reduced-motion and static fallback thinking.
- It is legible on mobile.
- It can move toward editable PPTX and Figma without being rebuilt from scratch.
